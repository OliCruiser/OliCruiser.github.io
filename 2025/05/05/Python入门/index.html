变量形式

1. 变量的作用域

  • 全局变量：是在函数外部定义的变量，整个程序都可以访问。
  • 局部变量：是在函数内部定义的变量，只能在该函数内部使用。

2. 全局变量

全局变量是在函数外定义的变量。它可以在程序的任何地方访问（包括函数内部）

3. 局部变量

局部变量是定义在函数内部的变量，只能在该函数内部访问。一旦函数执行完毕，局部变量就会消失。

4. 作用域与访问优先级

  • 函数内部可以访问全局变量，但修改全局变量需要用 global 关键字。
  • 函数内部不能直接修改全局变量，除非使用 global 声明它。

如果不使用 global，函数内部会创建一个新的局部变量，修改的是局部变量，而不是全局变量：

5. 为什么要用 global 关键字？

在有些场景下，我们希望函数内部能够修改全局变量的值。此时，使用 global 可以明确告诉 Python，这个变量是全局的，而不是局部的。

6. 局部变量和全局变量的总结

  • 全局变量：在函数外部定义，程序中的任何地方都可以访问。
  • 局部变量：在函数内部定义，只能在该函数内使用。

什么是类（class）？

类就像是一个“模具”，用来制造“产品”。比如说，如果你有一个“车”的模具，你可以用这个模具制造不同颜色、不同款式的车。

  • 类 就是这个“车”的模具。
  • 对象 就是根据模具做出来的“车”，每一辆车都可以有不同的颜色、款式（这些就是车的“属性”）。

类的构成

类里有两部分：

  1. 属性：就是每个对象的“特点”，比如“车”的颜色、型号。
  2. 方法：就是对象能做什么事情，比如“车”能开、能停。

浅复制（copy）与深复制（deepcopy）

在 Python 中，浅复制和深复制都涉及到如何复制对象，但它们的行为有很大的区别。

1. 浅复制（copy()）

  • 浅复制指的是创建一个新对象，但其中的元素是引用原对象中的元素。换句话说，浅复制不会创建被复制对象中元素的副本，而是仅仅复制了对象本身，元素依然指向原来的内存位置。
  • 浅复制适用于仅需要复制对象的外层结构，而不关心对象内部的嵌套结构（如列表中的列表、字典中的字典等）时。

  • original 和 shallow_copied 这两个列表是两个不同的对象，但它们包含的内部列表（如 [1, 2, 3] 和 [4, 5, 6]）是共享的。
  • 修改 original 中的一个内部列表，shallow_copied 中相同的内部列表也被修改了，因为它们引用的是同一个内存地址。

2. 深复制（deepcopy()）

  • 深复制会创建一个新对象，并且会递归地复制原对象中的所有内容，包括嵌套的元素。也就是说，深复制会复制所有层级的数据，避免了浅复制中可能出现的引用共享问题。
  • 深复制适用于需要完全独立的副本，不希望原对象和副本之间有任何联系的情况。

  • original 和 deep_copied 是完全独立的两个对象。

总结：

  • 浅复制（copy.copy()）：创建一个新的对象，但内部元素仍然是原对象中元素的引用。如果修改了原对象内部的嵌套对象，复制对象也会受到影响。
  • 深复制（copy.deepcopy()）：创建一个完全独立的新对象，包括原对象的嵌套对象都被递归地复制。这意味着修改原对象不会影响到复制对象，反之亦然。

何时使用？

  • 浅复制：适用于不需要关心嵌套元素独立性的场景。
  • 深复制：适用于需要完全独立副本的场景，尤其是对象包含嵌套对象时。

  1. **plot(x, y)**：绘制折线图。
  2. **xlabel() 和 ylabel()**：分别给 x 轴和 y 轴加上标签。
  3. **title()**：给图表加上标题。
  4. **show()**：显示图形。
  5. **grid(True)**：在图表上显示网格。

通过这几个简单的步骤，你就能轻松地把数据转化为图形，帮助你更直观地分析数据。如果你有任何问题，可以随时问我！

DQN

【深度强化学习】(1) DQN 模型解析，附Pytorch完整代码_dqn模型-CSDN博客

为什么：使用 Q 表存储每个状态 - 动作对的价值，当状态空间和动作空间维度很高时，Q 表会变得极其庞大

算法原理

基本原理

DQN（Deep Q Network）算法由 DeepMind 团队提出，是深度神经网络和 Q-Learning 算法相结合的一种基于价值的深度强化学习算法。

Q-Learning 算法构建了一个状态-动作值的 Q 表，其维度为 (s,a)，其中 s 是状态的数量，a 是动作的数量，根本上是 Q 表将状态和动作映射到 Q 值。此算法适用于状态数量能够计算的场景。但是在实际场景中，状态的数量可能很大，这使得构建 Q 表难以解决。为破除这一限制，我们使用 Q 函数来代替 Q 表的作用，后者将状态和动作映射到 Q 值的结果相同。
即得到一个函数，已知现在的状态，网络就会输出当前可执行的所有动作分别对应的 Q 值

模型结构

DQN 体系结构主要包含：Q 网络、目标网络，以及经验回放组件。

  • Q 网络是经过训练以生成最佳状态-动作值的 agent（智能体）。用于估计当前状态下各个动作的 Q 值（即动作价值），是不断更新训练的网络。
  • 经验回放单元的作用是与环境交互，生成数据以训练 Q 网络。其结构与 Q 网络相同，但参数更新频率较低。目标网络的存在是为了提供一个相对稳定的目标值，避免训练过程中 Q 值估计的剧烈波动，有助于训练的稳定性。
  • 目标网络与 Q 网络在初始时是完全相同的。DQN 工作流程图如下

有个叫经验回放的 “小仓库”，小孙子（智能体）在生活里（环境）做各种事，得到的经验（训练样本）就都存到这个 “小仓库” 里。

然后呢，从这个 “小仓库” 里随便拿出一些经验，给到两个 “小老师”，一个叫 Q 网络老师，一个叫目标网络老师。

目标网络老师就像个有经验的老教师，它给出一个做事的标准，这个标准就叫目标值。比如说，小孙子扫地，目标网络老师觉得扫干净这个动作应该得 8 分（目标值）。

Q 网络老师呢，是个正在学习怎么教人的新老师，它会自己估计小孙子扫地这个动作能得几分，比如它估计得 6 分。

这两个分数不一样，它们之间的差距就叫损失。咱得让 Q 网络老师变得更厉害，就根据这个损失，告诉 Q 网络老师哪里估计得不对，让它改。

但是呢，怕 Q 网络老师改得太乱，目标网络老师就时不时地把自己的教学方法（权重值）给 Q 网络老师参考一下，让 Q 网络老师能慢慢教得和目标网络老师差不多好。这样一直重复，直到小孙子学会把事情做好。

经验回放

经验回放从当前状态中以贪婪策略 选择一个动作，执行后从环境中获得奖励和下一步的状态，如下图所示。

与 Q Learning 算法不同，经验回放组件的存在有其必须性。神经网络通常接受一批数据，如果我们用单个样本去训练它，每个样本和相应的梯度将具有很大的方差，并且会导致网络权重永远不会收敛。

当我们训练神经网络时，最好的做法是在随机打乱的训练数据中选择一批样本。这确保了训练数据有足够的多样性，使网络能够学习有意义的权重，这些权重可以很好地泛化并且可以处理一系列数据值。如果我们以顺序动作传递一批数据，则不会达到此效果。

所以可得出结论：顺序操作彼此高度相关，并且不会像网络所希望的那样随机洗牌。这导致了一个 “灾难性遗忘” 的问题，网络忘记了它不久前学到的东西。

以上是引入经验回放组件的原因。智能体在内存容量范围内从一开始就执行的所有动作和观察都将被存储。然后从此存储器中随机选择一批样本。这确保了批次是经过打乱，并且包含来自旧样品和较新样品的足够多样性，这样能保证训练过的网络具有能处理所有场景的权重。

Q网络预测Q值

所有之前的经验回放都将保存为训练数据。现在从此训练数据中随机抽取一批样本，以便它包含较旧样本和较新样本的混合。随后将这批训练数据输入到两个网络。Q 网络从每个数据样本中获取当前状态和操作，并预测该特定操作的 Q 值，这是“预测 Q 值”。如下图所示。

目标网络预测目标 Q 值

目标网络从每个数据样本中获取下一个状态，并可以从该状态执行的所有操作中预测最佳 Q 值，这是“目标 Q 值”。如下图所示。

​ DQN 同时用到两个结构相同参数不同的神经网络，区别是一个用于训练，另一个不会在短期内得到训练，这样设置是从考虑实际效果出发的必然需求。

​ 如果构建具有单个 Q 网络且不存在目标网络的 DQN，假设此网络应该如下工作：通过 Q 网络执行两次传递，首先输出 “预测 Q 值”，然后输出 “目标 Q 值”。这可能会产生一个潜在的问题：Q 网络的权重在每个时间步长都会更新，从而改进了对“预测 Q 值”的预测。但是，由于网络及其权重相同，因此它也改变了我们预测的“目标 Q 值”的方向。它们不会保持稳定，在每次更新后可能会波动，类似一直追逐一个移动着的目标。

​ 通过采用第二个未经训练的网络，可以确保 “目标 Q 值” 至少在短时间内保持稳定。但这些“目标 Q 值”毕竟只是预测值，这是为改善它们的数值做出的妥协。所以在经过预先配置的时间步长后，需将 Q 网络中更新的权重复制到目标网络。

​ 可以得出，使用目标网络可以带来更稳定的训练。
